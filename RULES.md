# Task Completion Rules

1) Understand the request  
   - Clarify scope, deliverables, success criteria, and constraints.  
   - Identify existing components/data to reuse; avoid duplicates when possible.

2) Plan before editing  
   - Outline steps and dependencies; note risks and open questions.  
   - Choose modular, data-driven components; accept props for content/config.

3) Execute changes deliberately  
   - Prefer reusing or adapting existing sections; only build new when needed.  
   - Keep styling consistent with current design tokens/components.  
   - Add light comments only where non-obvious.

4) Validate  
   - Run relevant checks/tests or manual verification steps; note what was run.  
   - Re-read diffs for accuracy and unintended changes.

5) Document outcomes  
   - Summarize what changed and where (file paths).  
   - Call out any follow-ups, assumptions, or remaining risks.
